    Compress each image to target under 40KB while maintaining visual quality at their 20x20px display size:

    1. Use sharp or similar to resize each image to 40x40px (2x for retina displays) and compress:
       ```bash
       cd src/frontend
       npm install --save-dev sharp
       ```
       Then run a compression script:
       ```javascript
       // compress-icons.mjs
       import sharp from 'sharp';
       import { readdir, stat } from 'fs/promises';
       import { join } from 'path';

       const icons = [
         'dashboard.png', 'employees.png', 'attendance.png', 'payroll.png',
         'settings.png', 'oficial_reports.png', 'users_access.png', 'notification.png'
       ];

       for (const icon of icons) {
         const input = join('public/images/layout', icon);
         await sharp(input)
           .resize(40, 40, { fit: 'inside', withoutEnlargement: true })
           .png({ quality: 80, compressionLevel: 9 })
           .toFile(join('public/images/layout', icon.replace('.png', '.png')));
         console.log(`Compressed ${icon}`);
       }
       ```

    2. If sharp is not available or fails, use an alternative approach:
       - Use `pngquant` or `optipng` CLI tools
       - Or manually compress each image using an online tool like TinyPNG
       - Target: each icon under 40KB, total under 300KB

    3. Verify each compressed icon still looks correct at 20x20px display size.

    DO NOT change file names or paths — all existing next/Image references must continue to work.
