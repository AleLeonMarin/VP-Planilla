  <name>Task 1: Configure Next.js performance settings in next.config.ts</name>
  <files>src/frontend/next.config.ts</files>
  <action>
    Update `src/frontend/next.config.ts` with performance optimizations:

    ```typescript
    import type { NextConfig } from "next";

    const nextConfig: NextConfig = {
      compress: true,
      experimental: {
        optimizePackageImports: [
          '@heroicons/react',
          'lucide-react',
          '@fullcalendar/core',
          '@fullcalendar/react',
          'framer-motion',
        ],
      },
    };

    export default nextConfig;
    ```

    **Why these settings:**
    - `compress: true`: Enables gzip/Brotli compression for production builds. Reduces transfer size by 60-80%.
    - `optimizePackageImports`: Tree-shakes unused exports from large icon libraries. `@heroicons/react` has 300+ icons but the app uses ~30. `lucide-react` similar. `framer-motion` has motion components not all used.

    **Do NOT add:**
    - `images.unoptimized: true` — would break Next.js Image optimization
    - `reactStrictMode: true` — already default in Next.js 15
    - Any webpack config overrides — keep it simple
  </action>
  <verify>
    <automated>npx tsc --noEmit --project src/frontend/tsconfig.json 2>&1 | head -10</automated>
  </verify>
  <done>next.config.ts has compress:true and optimizePackageImports for @heroicons/react, lucide-react, @fullcalendar, framer-motion. TypeScript passes.</done>
</task>

  <files>src/frontend/src/app/layout.tsx</files>
  <action>
    Update `src/frontend/src/app/layout.tsx` to add font optimization hints:

    1. Add `<link>` tags in the `<head>` via metadata or directly in the HTML:
       ```tsx
       export const metadata = {
         title: { default: "VP Planillas", template: "%s | VP Planillas" },
         description: "Sistema integral de gestión de nómina y recursos humanos",
         icons: { icon: "/logo.ico", shortcut: "/logo.ico", apple: "/logo.ico" },
         other: {
           'link': [
             { rel: 'preload', href: '/fonts/VerdeFont.woff', as: 'font', type: 'font/woff', crossOrigin: 'anonymous' },
             { rel: 'preload', href: '/fonts/PraderaFont.woff', as: 'font', type: 'font/woff', crossOrigin: 'anonymous' },
           ],
         },
       };
       ```

    2. Verify `globals.css` already has `font-display: swap` on both @font-face declarations (confirmed from context — already correct, no changes needed).

    **Why preload:** The custom fonts (VerdeFont, PraderaFont) are used on every page. Preloading ensures the browser discovers them early in the critical rendering path, reducing FOIT (Flash of Invisible Text). Combined with `font-display: swap`, text is visible immediately with fallback font, then swaps when custom font loads.

    **Do NOT:**
    - Switch to `next/font` — the custom brand fonts are local .woff files, not Google Fonts. next/font is for Google/external fonts.
    - Remove the @font-face declarations from globals.css — preload complements them, doesn't replace them.
  </action>
