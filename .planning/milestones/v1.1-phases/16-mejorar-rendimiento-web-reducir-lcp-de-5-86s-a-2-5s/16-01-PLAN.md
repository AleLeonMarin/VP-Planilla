  <files>src/frontend/src/components/LaborEventModal.tsx, src/frontend/src/components/EditEmployeeModal.tsx, src/frontend/src/components/AddEmployeeModal.tsx, src/frontend/src/components/PayrollCreateModal.tsx, src/frontend/src/components/ui/NotificationPanel.tsx</files>
  <action>
    Framer-motion is used in 5 component files. These are modals/panels that only render when open. Use `next/dynamic` to defer loading:

    1. In each modal component file, add at the top:
       ```typescript
       import dynamic from 'next/dynamic';
       const MotionDiv = dynamic(() => import('framer-motion').then(mod => mod.motion.div), { ssr: false });
       const AnimatePresence = dynamic(() => import('framer-motion').then(mod => mod.AnimatePresence), { ssr: false });
       ```

    2. Replace `import { motion, AnimatePresence } from 'framer-motion'` with the dynamic imports above.

    3. Replace `<motion.div` with `<MotionDiv` and `<AnimatePresence` with the dynamic version.

    4. For `LaborEventModal.tsx` which also uses `useDragControls`: keep `import { useDragControls } from 'framer-motion'` as static (hooks cannot be dynamic). The hook is only called when the modal renders, and the modal itself is conditionally rendered by the parent.

    5. For `NotificationPanel.tsx`: Same pattern — dynamic import for `AnimatePresence` and `motion.div`.

    **Why not lazy-load the entire modal component:** The parent pages conditionally render modals based on state (`isOpen`). The framer-motion code is already tree-shakeable when modals are closed, but the import still adds to the bundle. Dynamic importing the motion primitives ensures the library code is only fetched when a modal actually opens.

    **Alternative considered:** Lazy-load entire modal components from their parent pages. Rejected because it would require changes to 10+ parent page files. Dynamic importing framer-motion primitives is a single-file change per modal.
  </action>
