<objective>
Fix loading and error state handling in positions and bonuses hooks — separate initial fetch loading from mutation loading, and ensure error states are properly detected.

Purpose: UAT found two issues: (1) skeletons appear during CRUD mutations because `isLoading` is set to true for both fetch and mutations, replacing existing data with skeleton rows. (2) Positions page shows empty state instead of error banner when backend is unavailable because the hook doesn't distinguish between fetch failure and empty data.

Output: Two hooks with separated `isFetching`/`isMutating` state, positions service verified to throw on network failures.
</objective>
