## Outstanding Issues


### Issue 1 - MISSING: Migration file for user_last_login (REQ 5.3, Must)

**Severity:** Medium - functional in dev; breaks production deploy
**Location:** src/backend/prisma/ - no migrations/ directory exists
**Details:** prisma db push was used instead of prisma migrate dev --name add_last_login_to_users due to DB drift. The user_last_login field is present in schema.prisma:302 and the Prisma client is generated correctly. Running prisma migrate deploy in a new environment will not apply this column. AuthService.updateLastLogin will fail at runtime outside the current dev DB.
**Fix:** Run npx prisma migrate dev --name add_last_login_to_users in a clean dev environment with DATABASE_URL pointing to a DB that does not yet have the column.

### Issue 2 - MISSING: Integration tests for POST /api/nominee/payroll (REQ 8.8, Should)

**Severity:** Low - Should priority; unit tests are comprehensive
**Details:** No supertest or HTTP-level test exists anywhere in src/backend/src/__tests__/. All 42 tests are unit tests. The endpoint is tested indirectly through NomineeService unit tests, but the full HTTP stack is not exercised.
**Fix:** Add src/backend/src/__tests__/integration/nominee.test.ts using supertest to cover POST /api/nominee/calculate-payroll.
