## Milestone Goal

Llevar VP-Planilla de "funcional pero frágil" a estable, seguro y completo. El sistema ya calculaba planillas correctamente — este milestone cerró los huecos de seguridad, eliminó bugs silenciosos, completó funcionalidad faltante y agregó cobertura de tests para el core del negocio.

---

## Phases

### Phase 1 — Singleton Prisma
**Goal:** Eliminar 16 instancias separadas de `new PrismaClient()` — todos los servicios usan el singleton de `lib/prisma.ts`
**Plans:** 3/3 complete
**Outcome:** 0 instancias sueltas, 1 connection pool compartido

### Phase 2 — Seguridad de Autenticación
**Goal:** Cerrar huecos críticos de auth — 13 rutas desprotegidas, JWT hardcodeado, credenciales en query params, throw undefined en PayrollService
**Plans:** 2/2 complete
**Outcome:** Todas las rutas protegidas, servidor falla si falta JWT_SECRET, login limpio

### Phase 3 — Validación de Inputs y CORS
**Goal:** Ningún `req.body` llega a Prisma sin validación Zod. CORS restringido.
**Plans:** 2/2 complete
**Outcome:** 5 schemas Zod, validateBody middleware, CORS con ALLOWED_ORIGINS

### Phase 4 — Performance del Cálculo de Planilla
**Goal:** `calculatePayrollForPeriod` ejecuta O(1) queries en lugar de O(N)
**Plans:** 2/2 complete
**Outcome:** Promise.all preload — 6 queries para cualquier número de empleados

### Phase 5 — Funcionalidad de Negocio Faltante
**Goal:** Ruta deprecated eliminada, updateLastLogin en DB, audit logs en operaciones críticas
**Plans:** 3/3 complete
**Outcome:** /api/nominee/calculate removido, last_login actualizado en login, audit logs en 3 operaciones

### Phase 6 — Feriados Nacionales Costa Rica
**Goal:** Los días feriados CR no cuentan como días laborales en el cálculo de planilla
**Plans:** 3/3 complete
**Outcome:** FERIADOS_CR lista estática, countWorkingDaysInPeriod excluye feriados, tests unitarios

### Phase 7 — Rate Limiting, Helmet y Token Revocation
**Goal:** Protección anti-fuerza bruta, headers de seguridad HTTP, logout real
**Plans:** 4/4 complete
**Outcome:** 10 req/15min/IP en login, 11 headers Helmet, vpg_token_blocklist para logout real

### Phase 8 — Tests Unitarios NomineeService
**Goal:** Cobertura de tests para todos los casos de cálculo de planilla
**Plans:** 2/2 complete
**Outcome:** 42 unit tests (regular, OT 1.5×, OT 2×, descanso trabajado, feriado CR, CCSS) + 3 integration tests

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Singleton Prisma via lib/prisma.ts | Ya existía, solo faltaba usarlo | ✅ Implementado Phase 1 |
| AuthMiddleware por ruta (router.use) | Más control que global; sin allowlist complicada | ✅ Implementado Phase 2 |
| Zod validation en backend | Frontend ya usa Zod — patrones familiares | ✅ Implementado Phase 3 |
| Feriados CR como lista estática | No hay API pública confiable | ✅ Implementado Phase 6 |
| Token revocation con DB blocklist | Redis no está en el stack | ✅ Implementado Phase 7 |
| No eliminar empleados — solo desactivar | Eliminar rompería historial de planillas | ✅ Regla de negocio documentada |
| prisma db push para last_login (dev) | DB drift impedía migrate dev limpio | ⚠️ Baseline migration 0_init creada |
