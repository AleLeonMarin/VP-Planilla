<objective>
Implement the backend security gate for critical legal parameter modifications. Any param with isCritical=true requires the caller to supply a confirmationPassword in the request body. The controller verifies the password via AuthService before delegating to LegalParamService. The service atomically upserts the param and writes an audit log entry containing full change metadata.

Purpose: Enforce step-up authentication at the API layer so critical legal values (overtime factors, CCSS rates, etc.) cannot be silently altered by a logged-in user without re-proving their identity.
Output: Updated AuthService, LegalParamController, and LegalParamService with Prisma transaction wrapping and audit trail.
</objective>

```typescript
/**
 * Verifies that the supplied plain-text password matches the stored hash for the given user.
 * Used to enforce step-up authentication before mutating critical legal parameters.
 *
 * @param userId - The numeric user ID whose password to verify
 * @param plainPassword - The password supplied by the caller (never logged)
 * @returns true if the password matches, false otherwise (including if user not found)
 * @throws Never — all DB errors are caught and return false to avoid leaking user existence
 */
static async verifyPasswordForUser(userId: string, plainPassword: string): Promise<boolean> {
  try {
    const user = await prisma.vpg_users.findUnique({
      where: { user_id: parseInt(userId, 10) },
      select: { user_password_hash: true },
    });
    if (!user?.user_password_hash) return false;
    return await bcrypt.compare(plainPassword, user.user_password_hash);
  } catch {
    // Do NOT include plainPassword in any error log
    return false;
  }
}
```

```typescript
static async upsertParam(data: UpsertParamDto, options: { passwordVerified?: boolean; userId?: number } = {}): Promise<LegalParam> {

  return await prisma.$transaction(async (tx) => {
    // Fetch current value for oldValue in audit log
    const existing = await tx.vpg_legal_params.findUnique({
      where: { legal_params_key: data.key },
      select: { legal_params_value: true },
    });
    const oldValue = existing?.legal_params_value ?? null;

    // Perform the upsert
    const result = await tx.vpg_legal_params.upsert({
      where: { legal_params_key: data.key },
      update: { /* existing update fields */ },
      create: { /* existing create fields */ },
    });

    // Write audit log — only when passwordVerified or when called without confirmation (non-critical)
    if (userId) {
      const auditDetails = JSON.stringify({
        paramKey: data.key,
        oldValue,
        newValue: data.value,
        source_decree: data.source_decree ?? null,
        password_confirmed: passwordVerified,
      });

        data: {
          audit_logs_user_id: userId,
          audit_logs_action: 'LEGAL_PARAM_UPDATE',
          audit_logs_entity: 'vpg_legal_params',
          audit_logs_entity_id: result.legal_params_id,
          audit_logs_timestamp: new Date(),

}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request body → Controller | confirmationPassword is an untrusted string from the client |
| Controller → AuthService | Plain password is passed in-process only; never persisted or logged |
| AuthService → Prisma | Read-only query for password hash; hash never returned to controller |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-62-01 | Spoofing | LegalParamController | mitigate | JWT token verified by AuthMiddleware.verifyToken before controller runs; userId extracted from verified token, not from request body |
| T-62-02 | Tampering | LegalParamService.upsertParam | mitigate | passwordVerified flag is set by the controller after successful bcrypt.compare — callers cannot inject a true flag without going through AuthService |
| T-62-03 | Information Disclosure | AuthService.verifyPasswordForUser | mitigate | Plain password is never logged, never included in error messages, never persisted; only the boolean result is propagated |
| T-62-04 | Elevation of Privilege | LegalParamController | mitigate | Returns 400 (no password) and 403 (wrong password) before reaching the service — critical params cannot be updated without explicit password confirmation |
| T-62-05 | Repudiation | vpg_audit_logs | mitigate | Every successful critical param change writes an audit row with password_confirmed:true, timestamp, userId, paramKey, old and new values inside a Prisma transaction — if the write fails, the upsert also rolls back |
</threat_model>
